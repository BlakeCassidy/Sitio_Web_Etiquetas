¿Por qué son importantes las etiquetas semánticas?

Las etiquetas semánticas en HTML5 le dan **significado** a la estructura de tu contenido, tanto para los navegadores como para los desarrolladores y las tecnologías de asistencia (como los lectores de pantalla). A diferencia de las etiquetas no semánticas como `<div>` o `<span>`, las etiquetas semánticas describen claramente el tipo de contenido que contienen, mejorando la **accesibilidad**, el **SEO** (optimización para motores de búsqueda) y la **legibilidad** del código.

A continuación, explicaremos algunas de las más utilizadas:

1. <header>

La etiqueta **`<header>`** representa el contenido introductorio de una sección o de todo el documento. A menudo contiene un grupo de ayudas de navegación, como un logo, el título del sitio y el menú principal. Piensa en ello como el "encabezado" o "cabecera" de una página o una sección específica.

2. <nav>

La etiqueta **`<nav>`** se utiliza para definir un grupo de enlaces de navegación. Estos son los enlaces principales que permiten a los usuarios moverse entre las diferentes páginas o secciones importantes de un sitio web. No todos los grupos de enlaces necesitan ser `<nav>`, solo los que tienen una función de navegación principal.

3. <main>

La etiqueta **`<main>`** representa el contenido **dominante** del `<body>` de un documento. Debe ser el contenido principal y central del documento, y debe ser único. Esto significa que no debe haber más de un elemento `<main>` en un documento, y su contenido no debe repetirse en otras partes del sitio (como en la barra lateral o el pie de página).

4. <section>

La etiqueta **`<section>`** se utiliza para agrupar contenido temáticamente relacionado. Piensa en ella como un "capítulo" o una "sección" dentro de un documento. Cada `<section>` debe tener generalmente un encabezado (como `<h1>` a `<h6>`) que defina el tema de esa sección. No es un contenedor genérico como `<div>`; debe tener un propósito temático.

5. <article>

La etiqueta **`<article>`** representa un componente de contenido independiente y autosuficiente. Esto significa que el contenido dentro de un `<article>` podría ser distribuido o reutilizado por sí mismo (por ejemplo, a través de un feed RSS o en otra página web) sin perder su significado. Ejemplos comunes incluyen entradas de blog, comentarios de usuario, artículos de noticias o productos individuales.
